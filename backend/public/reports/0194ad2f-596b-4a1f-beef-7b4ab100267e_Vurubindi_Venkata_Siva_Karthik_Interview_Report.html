📚 What a Good Answer Looks Like
The CSS box model describes the four layers around every element (outside to inside): Margin (space outside the border, transparent) → Border (visible boundary around the element) → Padding (space between border and content, inherits background) → Content (text, images). Key practical point: by default, width applies to the content box only — border and padding are added on top. Setting box-sizing: border-box includes border and padding within the declared width, making layouts predictable. This is why most CSS resets include * { box-sizing: border-box }.

What are pure and impure functions in JavaScript? Call by value vs call by reference?

~ Partial — Call by value/reference correctly explained; pure/impure function definitions incorrect
📚 What a Good Answer Looks Like
Pure function: always returns the same output for the same inputs AND has no side effects (no modifying external state, no API calls, no DOM mutations, no logging). Example: const add = (a, b) => a + b — calling it 1,000 times with (2,3) always returns 5. Impure function: either depends on external state (so output varies for same inputs) or produces side effects. Example: let count = 0; const increment = () => ++count — modifies external state. Functions that call other functions can be pure or impure — the distinction is about side effects and determinism, not nesting.

📚 Correct Complexity Explanation
Min-heap of size K: insert each element — if heap size exceeds K, pop the minimum. Final heap contains the top K elements. Time: O(N log K) — each of N elements may trigger a heap operation costing O(log K). Space: O(K). This is better than sorting all N elements (O(N log N)) when K is small. Quickselect: average O(N) time, O(1) space — finds the Kth largest in one pass on average but O(N²) worst case.

📚 Complete Approach
