📚 What a Good Answer Looks Like
Trailing zeros come from factors of 10 = 2 × 5. Since factors of 2 always exceed factors of 5, count only factors of 5 in N!. Formula: count = ⌊N/5⌋ + ⌊N/25⌋ + ⌊N/125⌋ + ... until the term = 0. Each term counts numbers contributing that power of 5 (e.g., 25 = 5², contributes two). Time: O(log N), Space: O(1). Never compute the factorial.

✗ Not Solved — Confused "divisible by 2" with "power of 2"; bit manipulation approach not known (Rating: 0/5)
📚 What a Good Answer Looks Like
Powers of 2 in binary: 2=10, 4=100, 8=1000 — exactly one bit is set. N-1 flips all bits below the set bit: 8-1=0111. So (n & (n-1)) == 0 and n > 0 iff N is a power of 2. Time: O(1), Space: O(1). The key conceptual distinction: "divisible by 2" means the last bit is 0 (even number). "Power of 2" means exactly one bit is 1 anywhere. These are completely different conditions — understand this difference clearly.

📚 What a Good Answer Looks Like
Key insight: at time T, bus i completes exactly ⌊T/time[i]⌋ trips. Total trips at time T = sum(T // time[i] for i in range(n)). This function is monotonically non-decreasing in T. Binary Search on T: search range [1, min(time) × totalTrips]. If total_trips(mid) >= target, try smaller: high = mid. Else: low = mid + 1. Time: O(N × log(min × target)). The floor division is the core of this problem — practise it on simpler problems first.

📚 What a Good Answer Looks Like
BFS on positions: use a queue of (position, steps). Start: enqueue (0, 0). For each dequeued position p: try p+a (forward) and p-b (backward if p-b >= 0). Skip if forbidden, already visited, or beyond safe upper bound (max(forbidden) + a + b). Return steps when p == x. BFS guarantees minimum steps because it explores positions level by level. Key: knowing that "BFS = minimum steps" is not enough — you must know how to model the graph (what are nodes, what are valid edges, what terminates). Practise 5 BFS problems to build this instinct.
