📚 What a Good Answer Looks Like
Use two pointers: slow moves 1 step, fast moves 2 steps. If fast reaches null (or fast.next is null) → no cycle. If slow == fast at any point → cycle detected. Why they meet: in a cycle, the fast pointer closes the gap by 1 each iteration — they must eventually be at the same node. Time: O(N), Space: O(1).

📚 What a Good Answer Looks Like
O(N) — Prefix Sum + Hashmap: maintain a running prefix sum and a hashmap of {prefix_sum: count}. At each index, check if prefix_sum - k exists in the map — if yes, add that count to result. Then add current prefix sum to the map. Why it works: a subarray from index i to j has sum k iff prefix[j] - prefix[i-1] = k. The hashmap lets you check this in O(1) for all previous i. Time: O(N), Space: O(N). The O(N²) brute force (two nested loops computing all subarray sums) is also valid as a starting point — learn it first, then the prefix sum optimisation.
