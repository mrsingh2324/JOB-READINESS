📚 What a Good Answer Looks Like
XOR Swap (O(1), no extra space): a ^= b; b ^= a; a ^= b. Uses XOR properties: a ^ a = 0 and a ^ 0 = a. Each step encodes both values into a single variable. Arithmetic Swap: a = a + b; b = a - b; a = a - b. After step 1, a holds the sum. Step 2 recovers original a as sum - b. Step 3 recovers original b. Both work in O(1) time and space with no temp variable.

Initial answer: "using low and high binary search" — not applicable to a stream since elements arrive one at a time in unknown order When pushed for more detail: "no idea ma'am" — complete unknown The two-heap approach (max-heap for lower half, min-heap for upper half) was not known
✗ Not Answered — Binary search suggestion not applicable; two-heap approach not known (Rating: 0/5)
📚 What a Good Answer Looks Like
Use two heaps: max-heap (left) stores the smaller half, min-heap (right) stores the larger half. Keep sizes balanced (differ by at most 1). On insert: add to max-heap, rebalance if needed. Median = top of max-heap (odd count) or average of both tops (even count). Insert: O(log N), Find median: O(1).

📚 What a Good Answer Looks Like
