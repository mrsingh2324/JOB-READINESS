📚 What a Good Answer Looks Like
Use two stacks: stack1 (for enqueue) and stack2 (for dequeue). Enqueue: push to stack1. Dequeue: if stack2 is empty, pop all elements from stack1 into stack2 (this reverses order); then pop from stack2. This gives amortised O(1) for both operations. The key insight: reversing via a second stack converts LIFO to FIFO. This is a classic must-know interview problem.
Q2: How can you merge multiple sorted data streams?
Immediately responded: "I have no idea about data streams" — did not attempt the question at all No partial approach or alternate framing offered
✗ Not Answered — No knowledge of data streams or merge strategies (Rating: 0/5)
📚 What a Good Answer Looks Like
Use a Min-Heap (Priority Queue). Insert the first element of each stream into the heap along with its stream index and element index. Repeatedly extract the minimum, add it to the result, and insert the next element from that stream. Repeat until all streams are exhausted. Time: O(N log K) where N = total elements, K = number of streams. Space: O(K) for the heap. This is the optimal merge strategy for K sorted streams.

📚 What a Good Answer Looks Like
Greedy O(N): iterate through the array tracking max_so_far. At each index i, if max_so_far == i, a valid chunk boundary exists — increment chunk count. Why it works: if the maximum element seen so far equals the current index, all elements in this chunk are a permutation of [0..i] and can be sorted in place without affecting elements outside. Time: O(N), Space: O(1).
