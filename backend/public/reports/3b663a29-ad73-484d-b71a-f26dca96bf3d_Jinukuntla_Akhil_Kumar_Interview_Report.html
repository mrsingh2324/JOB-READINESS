Q1: Count the trailing zeros in the factorial of N. What is the optimal approach?
Initial approach: calculate the factorial of N, then use modulo operations to count trailing zeros — correct intuition but not feasible for large N Interviewer correctly pointed out: factorial of N for large values (100, 1000, 100000) cannot be stored in any standard data type (int, long, etc.) — overflow makes this approach impossible Could not arrive at the mathematical insight: trailing zeros come only from factors of 10 = 2 × 5; since factors of 2 always exceed factors of 5 in a factorial, counting factors of 5 gives the answer Proposed recursion for computing factorial — further from the optimal, not closer
~ Partial — Intuition correct but infeasible for large N; optimal O(log N) formula not reached (Rating: 2/5)
📚 What a Good Answer Looks Like
Trailing zeros in N! = count of pairs (2 × 5) in the factorial. Since factors of 2 always exceed factors of 5, count only factors of 5. Formula: count = floor(N/5) + floor(N/25) + floor(N/125) + ... until N/5^k = 0. Each term counts multiples of 5, 25, 125 etc. (25 contributes two factors of 5, 125 contributes three, etc.). Time: O(log N), Space: O(1). Never compute the factorial itself.
Q2: Check if a number is a power of 2. What is the most optimal approach?
Brute force: start from 2, multiply by 2 in a loop, compare with N — O(log N). Correct but not optimal Second approach: continuously mod by 2, check if remainder is 0 — same O(log N) complexity, just a different framing of the same idea Could not identify the O(1) bit manipulation trick — interviewer confirmed the optimal solution takes O(1) time
✗ Brute Force Only — O(log N) approach identified; O(1) bit manipulation solution not known (Rating: 1/5)
📚 What a Good Answer Looks Like
A power of 2 in binary has exactly one bit set: 2=10, 4=100, 8=1000. Subtracting 1 flips all lower bits: 8-1=7=0111. Therefore (n & (n-1)) == 0 and n > 0 iff n is a power of 2. Time: O(1), Space: O(1). This is one of the most important bit manipulation patterns — also handles edge case n=0 (0 & -1 = 0 but 0 is not a power of 2, hence the n > 0 check). Must-know for interviews.
