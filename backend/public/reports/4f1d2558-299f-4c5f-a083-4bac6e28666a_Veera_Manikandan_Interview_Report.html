Rapid Fire — DSA Theory
Find the majority element in an array (appears more than n/2 times) — most efficient approach
Initial approach: count frequency for each element — O(n²) or O(n) with a dictionary — functionally correct When prompted for O(1) space, moved toward Boyer-Moore Voting Algorithm concept: maintain count and candidate, reset when count reaches zero Could not articulate the Boyer-Moore approach cleanly by name but demonstrated the core intuition
~ Partial — Frequency approach correct; O(1) space Boyer-Moore reached only partially (Rating: 4/5)
📚 What a Good Answer Looks Like
Boyer-Moore Voting Algorithm: initialise candidate = None, count = 0. For each element: if count == 0, set candidate = num. If num == candidate, count += 1, else count -= 1. The final candidate is the majority element. Time O(n), Space O(1). Works because majority element always out-votes all others combined.
Detect a cycle in a linked list
Approach given: traverse the list and store visited nodes in a set; if a node is already in the set, a cycle exists Correct O(n) time, O(n) space approach — valid but not the most space-efficient Did not mention Floyd's Cycle Detection (slow/fast pointers) which achieves O(1) space
~ Partial — HashSet approach correct; Floyd's O(1) space approach not mentioned (Rating: 3/5)
📚 What a Good Answer Looks Like
Floyd's Cycle Detection (Tortoise & Hare): use two pointers — slow (moves 1 step) and fast (moves 2 steps). If they meet, a cycle exists. If fast or fast.next becomes None, no cycle. Time O(n), Space O(1) — optimal. HashSet approach is also valid (O(n) space) and worth mentioning as the simpler alternative.

Approach & Thought Process

~ Partial — Approach directionally correct; expansion logic had a gap (only explored j+1, missed i+1 pairs) (Rating: 4/5)
📚 What a Good Answer Looks Like
Initialise min-heap with (nums1[i] + nums2[0], i, 0) for all i in range(min(k, len(nums1))). Pop min, add pair to result. Push (nums1[i] + nums2[j+1], i, j+1) if j+1 < len(nums2). This correctly explores the next best pair from each nums1 row. Repeat k times. Time: O(k log k), Space: O(k). The interviewer noted: you need to also consider whether the next smallest could come from advancing i — but since all nums1[i]+nums2[0] are seeded initially, advancing j from each i is sufficient.

📚 What a Good Answer Looks Like
Arrow functions (const fn = (x) => x * 2) are a concise function syntax introduced in ES6. Key differences from regular functions: (1) No own this — they inherit this from the enclosing lexical scope (solves callback this binding issues). (2) No arguments object. They are NOT inherently async — you can make one async with const fn = async () => await fetch(...).

📚 What a Good Answer Looks Like
Node.js has a built-in http module: http.createServer((req, res) => {...}). Express wraps this with: (1) Router — maps URL patterns to handlers (app.get('/users', handler)). (2) Middleware — functions that process requests before they reach route handlers (app.use(express.json())). (3) Simplified request/response API — res.json(), req.body, req.params instead of raw stream parsing. Express is essentially a thin, powerful abstraction over Node's raw HTTP server.

📚 What a Good Answer Looks Like
MongoDB indexes use a B-Tree data structure to avoid full collection scans. Create with: db.users.createIndex({ email: 1 }). Types: single field, compound, text, geospatial. Always index fields in find(), sort(), and join ($lookup) operations.
