Coding Problem 1 — Number of Matching Subsequences ↗
Approach & Thought Process
First attempt: construct all substrings from the given string, check if each word is among them — exponential complexity (2^n) When pushed for better: iterated over the words array, checking if each word can be formed from the stream string — O(|words| × |s|) approach Correctly articulated the improved time complexity Did not arrive at the optimal bucket/pointer approach independently
~ Partial — Mid-level approach; improved from exponential but not optimal (Rating: 3/5)
📚 What a Good Answer Looks Like
Optimal: bucket approach. Group words by their first character into 26 buckets. For each character in s: look at the bucket for that character, advance each word's pointer, if pointer reaches end → count++, else move word to the bucket of its new first character. Time O(|s| + total chars in words), Space O(total chars). Alternatively: binary search on precomputed index lists for each character → O(|s| + |words| × len × log|s|).

Implemented the O(|words| × |s|) approach — passed approximately 32/56 test cases Code structure was correct for the approach taken; failed on edge cases and performance limits

Coding Problem 2 — Minimum Additions to Make Valid String ↗
Approach & Thought Process
Recognised the problem involves only characters a, b, c and the pattern "abc" must be formed Used a counter-based approach: count occurrences of each character; find the maximum count; add the difference for the other two characters Self-identified the flaw: this approach counts occurrences but ignores the order constraint — "abc" must appear in order Correctly articulated why the approach fails for ordered patterns
~ Partial — Basic approach attempted; correctly identified why it fails (order not preserved) (Rating: 3/5)
📚 What a Good Answer Looks Like
Greedy approach: iterate through the string maintaining a pointer j into "abc". For each character: if it matches "abc"[j], advance j; if j reaches 3, reset to 0 (complete group formed). At the end, add the cost to complete the last partial group: j == 0 ? 0 : 3 - j. Time O(n), Space O(1). The key insight: process greedily — each character either extends the current "abc" group or starts a new one.

📚 What a Good Answer Looks Like
Virtual DOM: React keeps a lightweight copy of the DOM in memory. On state/prop change, React re-renders the virtual DOM, diffs it against the previous virtual DOM (reconciliation), and applies only the changed nodes to the real DOM. This is far more efficient than re-rendering the entire real DOM. The diffing algorithm (Fiber) runs in O(n) using heuristics. This is why setState triggers a re-render rather than direct mutation — React needs to control when and how the real DOM updates.
