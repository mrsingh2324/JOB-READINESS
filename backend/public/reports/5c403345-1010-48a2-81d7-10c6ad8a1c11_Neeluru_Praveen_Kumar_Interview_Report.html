📚 What a Good Answer Looks Like
LRU Cache: evict the Least Recently Used element when capacity is exceeded. Implementation: HashMap (O(1) lookup) + Doubly Linked List (O(1) insertion/deletion). Most recently used at head, least recently used at tail. In Python: collections.OrderedDict. Time: O(1) for both get and put. Median from Stream: maintain two heaps — a max-heap for the lower half and a min-heap for the upper half. Keep sizes balanced (differ by at most 1). Median = top of max-heap (odd count) or average of both tops (even count). Insert: O(log N), Find median: O(1).

📚 What a Good Answer Looks Like
Iterative BFS (avoids stack overflow): use a queue. Start: enqueue the start index, mark visited. For each dequeued index: if arr[index] == 0 → return True. Else enqueue arr[index] ± index if in bounds and not visited. If queue empty → return False. Why BFS over recursive DFS: recursive DFS can exceed Python's default recursion limit (~1000) for large N. Iterative BFS uses a heap-allocated queue with no depth limit. Both are O(N) time and O(N) space. Rule of thumb: for graph/tree problems with N > 10⁴, prefer iterative traversal.
