📚 What a Good Answer Looks Like
Stateless: each HTTP request is independent — the server stores no client session between requests. Every request must carry all necessary context (e.g. user ID, auth token). This enables horizontal scaling — any server instance can handle any request. Stateful (e.g. WebSockets): the server maintains an open connection and remembers context across messages. The identifier is implied by the connection — you don't resend it each time.

📚 What a Full Explanation Looks Like
Approach: maintain a running prefix sum and a hashmap {prefix_sum: count} initialised with {0: 1}. For each element: add it to prefix_sum. Check if prefix_sum - K exists in the hashmap — if yes, add its count to the result. Then increment hashmap[prefix_sum]. This works because any subarray sum equalling K corresponds to two prefix sums whose difference is K. Time: O(N) — single pass through the array. Space: O(N) — hashmap stores at most N distinct prefix sums. You must be able to explain this fully and independently in the re-attempt.
