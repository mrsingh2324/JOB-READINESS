📚 What a Good Answer Looks Like
Boyer-Moore Voting: maintain candidate = None and count = 0. For each element: if count == 0, set candidate = element. If element == candidate, increment count; else decrement. Return candidate. Why it works: the majority element outnumbers all others combined, so it always survives cancellations. Time: O(N), Space: O(1). This is one of the most commonly tested interview patterns — study and memorise it with full understanding.

📚 What a Good Answer Looks Like
XOR all elements: result = 0; for num in arr: result ^= num; return result a ^ a = 0 (pairs cancel), a ^ 0 = a (identity). All duplicates cancel, leaving only the unique element. Time: O(N), Space: O(1). Study this alongside Boyer-Moore — both are O(1) space classics that appear constantly in interviews.

✗ Not Solved — Problem understood; stack awareness exists but monotonic stack with index tracking not reached (Rating: 0/5)
📚 What a Good Answer Looks Like
Monotonic Decreasing Stack: push indices (not temperatures). For each index i: while stack is not empty and temp[i] > temp[stack.top()]: pop index j, set result[j] = i - j. Push i. Why push indices not temperatures: you need to compute the gap (i - j), which requires knowing where j was. The stack stays decreasing because once a warmer day is found, the waiting period is resolved. Time: O(N), Space: O(N). Practise: Next Greater Element I & II, Next Smaller Element — all use the same pattern.
Coding Problem 2 — Maximum Number of Jumps to Reach Last Index ↗
Approach & Thought Process
Understood: from index i, can jump to any index j (i < j) if |nums[i] - nums[j]| <= target; find the maximum number of jumps to reach the last index; return -1 if not reachable Initial approach: greedy — always jump to the farthest reachable index — interviewer correctly identified this is wrong (greedy maximises step size, not step count) Second attempt: described DP in vague terms — "track farthest and find maximum ranges" — not a clear formulation When interviewer directly described the recursive DP approach, honestly acknowledged: "I haven't done DP before" No code produced; session concluded
✗ Not Solved — Greedy approach incorrect; DP not studied; honest acknowledgement of the gap (Rating: 0/5)
📚 What a Good Answer Looks Like
1D DP: dp[i] = maximum jumps to reach index i. Initialise dp[0] = 0, all others = -1 (unreachable). For each i from 1 to n-1: for each j from 0 to i-1: if |nums[i] - nums[j]| <= target and dp[j] != -1: dp[i] = max(dp[i], dp[j] + 1). Return dp[n-1]. Time: O(N²), Space: O(N). This is a standard "maximum path" DP. Key insight: greedy does not work here because taking a large jump now may prevent more jumps later — you must explore all valid transitions.
