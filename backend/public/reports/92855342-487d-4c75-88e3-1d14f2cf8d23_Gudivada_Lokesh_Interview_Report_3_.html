📚 What a Good Answer Looks Like
Boyer-Moore Voting: maintain candidate = None and count = 0. For each element: if count == 0 → set candidate. If element matches candidate → increment count. Otherwise → decrement count. Return candidate. Time: O(N), Space: O(1). The majority element always survives because it outnumbers all others combined. Study this alongside XOR single element — both are O(1) space pattern questions that appear constantly in DSA rounds.

📚 What a Good Answer Looks Like
Virtual DOM: a lightweight JavaScript object representation of the real DOM kept in memory. When state changes, React builds a new Virtual DOM tree, compares it with the previous one (diffing), and applies only the minimal changes to the real DOM. This avoids expensive full page re-renders. Reconciliation: React's algorithm for comparing old and new Virtual DOM trees. Same element type → update props in place. Different type → unmount old component, mount new. Keys help React match list items efficiently across renders.

📚 What a Good Answer Looks Like
Pure function: always returns the same output for the same input and has no side effects (no modifying external state, no API calls, no DOM mutations). Example: const add = (a, b) => a + b is pure; const log = (x) => { console.log(x); return x; } is impure. Pure component (React.memo for functional components): only re-renders when its props change. Prevents unnecessary re-renders when a parent re-renders but the child's props are the same. Equivalent to shouldComponentUpdate with a shallow prop comparison in class components.

📚 What a Good Answer Looks Like
Primitives (number, string, boolean, null, undefined) are passed by value — a copy of the value is made; changes inside the function don't affect the original. Objects and arrays are passed by reference — the reference (memory address) is copied, so mutations inside the function affect the original object. Reassigning the parameter itself, however, does not affect the original.
