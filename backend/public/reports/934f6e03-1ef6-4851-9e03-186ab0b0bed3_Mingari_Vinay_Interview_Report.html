📚 What a Good Answer Looks Like
Boyer-Moore Voting: candidate = None, count = 0. For each element: if count == 0 → set candidate. If element == candidate → count += 1. Else → count -= 1. Return candidate. You already know XOR cancels duplicates (a^a=0). Boyer-Moore uses the same logic conceptually: the majority element "cancels" all minority elements and still survives. Time: O(N), Space: O(1). These two patterns are a natural pair — learn them together.

📚 What a Good Answer Looks Like
Monotonic Decreasing Stack: push indices (not temperatures). For each index i: while stack not empty and temp[i] > temp[stack.top()]: pop j, set result[j] = i - j. Push i. Return result. The critical insight: you never push all elements first. You process one element at a time — each new element resolves any previous unresolved days that are now warmer. Push indices (not values) because you need the gap i - j. Study this pattern until you can implement it without reference: it is one of the most common medium-level patterns.

📚 What a Good Answer Looks Like
1D DP: dp[i] = max jumps to reach index i. Init dp[0] = 0, all others = -1. For each i from 1 to n-1: for each j from 0 to i-1: if abs(nums[i] - nums[j]) <= target and dp[j] != -1: dp[i] = max(dp[i], dp[j] + 1). Return dp[n-1]. Time: O(N²), Space: O(N). The DP state is simple: maximum jumps to reach each index. If a valid jump exists from j to i and j was reachable, extend. Study Climbing Stairs and House Robber first to build DP intuition before attempting this problem.
