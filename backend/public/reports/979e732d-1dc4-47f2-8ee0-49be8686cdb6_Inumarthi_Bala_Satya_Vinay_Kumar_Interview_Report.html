📚 What a Good Answer Looks Like
O(N) time, O(1) space — Boyer-Moore Voting: maintain a candidate and count. For each element: if count = 0, set candidate = element. If element = candidate, increment count; else decrement count. The final candidate is the majority element. Why it works: the majority element appears more than all others combined, so it always "survives" the cancellation. Also solvable in O(N) time and O(N) space using a hashmap frequency count — know both approaches.

📚 What a Good Answer Looks Like
Use XOR: result = 0; for num in arr: result ^= num; return result XOR properties: a ^ a = 0 (pairs cancel) and a ^ 0 = a (identity). All duplicate pairs cancel, leaving only the unique element. Time: O(N), Space: O(1). This is a must-know bit manipulation pattern that appears very frequently in interviews.

📚 What a Good Answer Looks Like
Sort cars by position in descending order (closest to target first). Compute time for each car: time = (target - pos) / speed. Use a stack. For each car (in sorted order): if its time > the top of the stack, it cannot catch the car ahead — push it as a new fleet. Otherwise it merges with the fleet ahead — do not push. Answer = stack size. Time: O(N log N) for sorting. Space: O(N) for stack. The key insight: a faster car behind a slower one will always slow down to match — they form one fleet.

📚 What a Good Answer Looks Like
Track running prefix sum parity. Maintain odd_count and even_count (initialise even_count = 1 for empty prefix). For each element: update prefix parity. If current prefix is odd, add even_count to result (odd - even = odd). If even, add odd_count. Then increment the respective counter.
