📚 What a Complete Answer Looks Like
Optimal: use a hashmap. For each element num: check if target - num exists in the map. If yes, return the pair. If no, store map[num] = index. Time: O(N), Space: O(N) — no sorting needed. Two-pointer is also valid if sorted: O(N log N) time, O(1) space. Both are acceptable answers — mentioning both and comparing trade-offs would have scored full marks.
Q2: Every integer appears twice except one. Find the singly occurring element in O(N) time and O(1) space.
Proposed creating a frequency dictionary (hashmap) — iterate through all key-value pairs, return the key with count 1 When the interviewer pointed out this costs O(N) extra space (not O(1)), could not find a way to optimise Said explicitly: "I don't think I can optimise it in that way" — XOR approach not known
✗ Incomplete — Hashmap approach logically correct but violates O(1) space constraint; XOR trick not known (Rating: 0/5 for optimal)
📚 What a Good Answer Looks Like
XOR all elements: result = 0; for num in arr: result ^= num; return result a ^ a = 0 (duplicates cancel out), a ^ 0 = a (identity). All pairs cancel, leaving only the unique element. Time: O(N), Space: O(1). This is a must-know O(1) space trick for interviews.

📚 What a Good Answer Looks Like
Build prefix XOR array: prefix[0] = 0; prefix[i] = prefix[i-1] ^ arr[i-1]. For each query [L, R]: answer is prefix[R+1] ^ prefix[L]. Time: O(N + Q), Space: O(N). You already knew prefix sums — the only leap needed is that XOR replaces addition. Since a ^ a = 0, the prefix XOR trick cancels the same way prefix sums do.
