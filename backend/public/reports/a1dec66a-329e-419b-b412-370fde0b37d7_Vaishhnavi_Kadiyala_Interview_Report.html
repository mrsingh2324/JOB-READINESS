📚 What a Good Answer Looks Like
Use modified binary search. At each step, check which half is sorted: if arr[mid] >= arr[low], left half is sorted — check if target lies in it, else move right. Otherwise right half is sorted — check if target lies in it, else move left. Time complexity: O(log n). Space: O(1). Key insight: one half is always sorted in a once-rotated array.

📚 What a Good Answer Looks Like
Use two pointers: slow moves one step at a time, fast moves two steps. When fast reaches the end, slow is at the middle. For cycle detection: if slow == fast at any point, a cycle exists. Time: O(n), Space: O(1). This technique is foundational for linked list problems — Floyd's algorithm.

📚 What a Good Answer Looks Like
Use prefix sums + hashmap. For each index, compute prefix sum. Count of subarrays ending here with sum = goal equals prefix_count[prefix_sum - goal]. Store prefix sums in a hashmap as you iterate. Time: O(n), Space: O(n). For the binary array constraint, a sliding window approach also works in O(n) using the trick: atMost(goal) - atMost(goal - 1).
