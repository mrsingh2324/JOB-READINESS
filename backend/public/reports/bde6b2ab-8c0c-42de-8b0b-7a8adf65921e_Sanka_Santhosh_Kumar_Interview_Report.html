📚 What a Good Answer Looks Like
Boyer-Moore Voting: maintain candidate = None and count = 0. For each element: if count == 0 → set candidate. If element == candidate → count += 1. Else → count -= 1. Return candidate. Why it works: the majority element always survives cancellation because it outnumbers all others combined. Time: O(N), Space: O(1). Two pointers only works on sorted arrays — for unsorted frequency problems, Boyer-Moore is the correct O(1) tool.

📚 What a Good Answer Looks Like
Sort cars by position descending (farthest first). Compute time[i] = (target - pos[i]) / speed[i]. Iterate: maintain a stack of fleet times. If current car's time > stack top, it cannot catch the car ahead → new fleet, push to stack. Otherwise it joins the fleet ahead (discard). Fleet count = stack size at the end. Time: O(N log N) for sort, O(N) for traversal. The key insight: sorting farthest-first means you always know whether a car can catch the one ahead of it — if its time is greater, it cannot (it arrives later), so it's a new fleet.

📚 What a Good Answer Looks Like
1D DP: dp[i] = max jumps to reach index i. Init dp[0] = 0, all others = -1. For each i from 1 to n-1: for each j from 0 to i-1: if abs(nums[i] - nums[j]) <= target and dp[j] != -1: dp[i] = max(dp[i], dp[j] + 1). Return dp[n-1]. Time: O(N²), Space: O(N). Sliding window processes contiguous subarrays — it cannot evaluate all valid jump pairs. DP is needed because each state depends on all valid predecessors.
