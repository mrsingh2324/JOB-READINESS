📚 The Missing Binary Search Logic
In a sorted array with one unique element: pairs occupy (even, odd) index positions before the unique element and (odd, even) after. At index mid (even): if arr[mid] == arr[mid+1], unique is on the right → low = mid + 2. If arr[mid] != arr[mid+1], unique is at mid or left → high = mid. At index mid (odd): if arr[mid] == arr[mid-1], unique is on the right → low = mid + 1. Otherwise → high = mid - 1. Time: O(log N), Space: O(1).

📚 What a Good Answer Looks Like
Sort cars by position in descending order (closest to target first). Compute time[i] = (target - position[i]) / speed[i] for each car. Use a stack. For each car's time (in sorted order): if time[i] > stack top, this car cannot catch the one ahead — push as a new fleet. Otherwise it merges with the fleet ahead — do not push. Answer = stack size. Time: O(N log N) for sort.

📚 What a Good Answer Looks Like
BFS with index-based node numbering. Assign root index = 1. For a node at index i: left child = 2i, right child = 2i + 1. At each level, width = last node index - first node index + 1. Normalise by subtracting the first index at each level to prevent integer overflow. Use a queue of (node, index) pairs. For each level: record first and last index, compute width, enqueue children with their computed indices. Time: O(N), Space: O(N). The null gaps are automatically captured because non-existent nodes between valid ones still have valid computed indices.
