Greedy O(N): iterate tracking max_so_far. At each index i: if max_so_far == i, all elements in [0..i] are a permutation of [0..i] and can be independently sorted → increment chunk count. Approach & Thought Process
Correctly understood: the string must be decomposable into complete "abc" groups; find the minimum number of characters to add to make this possible Correctly traced specific examples: single "a" needs "bc" added (2 additions); "aaa" needs 2 additions per group = 6 total; "abc" needs 0 additions Proposed: condition-based approach — check which letter is given and handle each case separately — not generalised Could not reach the greedy pointer approach: iterate through the string maintaining a pointer into "abc", advance both when characters match, count a new group when the string pointer does not match Time complexity stated as O(2) — interviewer correctly explained O(2) = O(1) = constant; then corrected to acknowledge a loop is needed
~ Partial — Problem and examples correctly understood; generalised greedy pointer approach not reached (Rating: 2/5 PS)
📚 What a Good Answer Looks Like
Greedy pointer: maintain index j into "abc" (0=a, 1=b, 2=c). For each character in the input: if it matches "abc"[j], advance j. If j reaches 3, a complete group is formed — reset j=0. Count additions = (number of complete groups × 3) - len(string) + (2-j if j > 0 else 0). Equivalently: iterate through string with a pointer in "abc". When the current character breaks the expected sequence, it starts a new group. Additions = (groups × 3) - len(string). Time: O(N), Space: O(1).
