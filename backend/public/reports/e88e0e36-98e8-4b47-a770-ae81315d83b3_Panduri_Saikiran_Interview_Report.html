📚 What a Good Answer Looks Like
Trailing zeros in N! come only from factors of 10 = 2 × 5. Since factors of 2 always outnumber factors of 5 in a factorial, only count factors of 5. Formula: count = ⌊N/5⌋ + ⌊N/25⌋ + ⌊N/125⌋ + ... — each term counts numbers contributing that many factors of 5 (e.g., 25 contributes two). Time: O(log N), Space: O(1). Never compute the factorial itself.

📚 What a Good Answer Looks Like
Powers of 2 in binary: 2=10, 4=100, 8=1000, 16=10000 — exactly one bit set. Subtracting 1 flips all lower bits: 8-1 = 0111. Therefore (n & (n-1)) == 0 and n > 0 gives the answer. Time: O(1), Space: O(1). This is one of the most common bit manipulation patterns in interviews — also know: count set bits (n & (n-1) repeated), check kth bit (n & (1 << k)), XOR for single element.

📚 What a Good Answer Looks Like
BFS: treat each position as a node. From position p, edges go to p+a (if not forbidden and within bounds) and p-b (if not forbidden and >= 0 and we haven't used too many backward jumps). Use a visited set to avoid cycles. BFS guarantees the shortest path (minimum jumps). The key upper bound for search: positions beyond max(forbidden) + a + b never need to be visited. Time: O((max_pos) × 2), Space: O(max_pos). Recognising this as a shortest-path on an implicit graph is the critical pattern — any "minimum steps to reach target" problem with valid/invalid states is almost always BFS.
