📚 Indexing — What a Good Answer Looks Like
An index is a data structure (B-tree) built on one or more fields so MongoDB can find documents without scanning the full collection. Speeds up read queries dramatically. Trade-off: every write (insert/update/delete) must also update the index — so write-heavy collections should not be over-indexed. Example: db.users.createIndex({'{'} email: 1 {'}'}) makes all queries filtering by email much faster.

📚 What a Good Answer Looks Like — Product of Array Except Self (LeetCode 238)
Build two arrays: prefix[i] = product of all elements to the left of i; suffix[i] = product of all elements to the right of i. Answer for each index: prefix[i] * suffix[i]. Optimise to O(1) space by computing prefix in the output array, then multiplying suffix in a second pass using a running variable. Time: O(N), Space: O(1) extra. This problem tests whether you can think in terms of pre-computation and passes — the same mental model used in prefix sum, sliding window, and two-pointer problems. Study it fully: understand why division is excluded and why two passes solve it cleanly.
